현재 작업 상황을 work-logs에 기록합니다.

## 절차

1. `work-logs/` 디렉토리의 기존 로그 파일 확인
2. 현재 브랜치, 최근 커밋, 변경사항을 분석
3. 기존 로그 파일이 있고 같은 작업이면 업데이트, 새 작업이면 새 파일 생성

## 로그 파일 형식

파일명: `{마일스톤}-{번호}_{작업명}.md`

```markdown
# {마일스톤}-{번호}: {작업 제목}

- **날짜**: {YYYY-MM-DD}
- **마일스톤**: {M0/M1/...}
- **브랜치**: {브랜치명}
- **상태**: {진행 중 | 완료 | 블로커}

## 작업 내용

### 완료
- [x] 완료된 항목

### 진행 예정
- [ ] 남은 항목

## 커밋 이력
- {hash 7자리} {커밋 메시지}

## 메모
{특이사항, 결정 사항, 블로커 등}
```

기존 파일이 있으면 체크리스트와 커밋 이력만 업데이트합니다.
